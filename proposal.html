3. Tools to be Used :

Backend & Database:
• Node.js
• Oracle Database 23ai Express

4. Key Features :
  • i. Offline logging: Captures Sales & Stock receipt without active connection.
  • ii. Sync Queue: FIFO auto uploads local operations upon reconnection.
  • iii. Native Scan: Direct camera EAN barcode detector without third-party JS.
  • iv. Zero-library charts: Sales line metrics rendered natively via Canvas 2D.
2. Objective :

To design and implement a zero-dependency, offline-first stock management Single Page Application (SPA) tailored for micro-retail shops in Bangladesh.

The project aims to demonstrate advanced browser APIs (IndexedDB, Service Workers, Web Audio) integrated with an Oracle Database to keep inventory states fully synchronized and data transactions idempotent during network failures.

5. Target Audience :
  • Small-to-medium grocery shop cashiers and supervisors in connectivity-unstable regions.
  • Retailers seeking lightweight stock software with zero local installation overhead.
6. Expected Outcome :

A fully deployable, installable Progressive Web Application (PWA) that starts instantly, enables checkout, and syncs queue elements autonomously.

The final product establishes strict role-based permission tiers (Admin / Staff) and provides clean, error-resistant SQL triggers.
